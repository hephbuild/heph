# In-place codegen (`codegen = "in_place"`)

A formatter that rewrites its **own, committed** source files — and a contrasting
`codegen = "copy"` target that emits a **generated, gitignored** file.

Run everything below from the `example/` workspace root (the dir with
`.hephconfig2`). `heph` ≈ `cargo run -q --` with `HEPH_CWD` pointing here.

## In-place: `//fmt:fmt`

`greeting.txt` and `note.txt` are tracked, lowercase source. The `fmt` target
"formats" them (here: uppercases) **in place**.

```
$ heph run //fmt:fmt
$ cat fmt/greeting.txt
HELLO WORLD
```

Three things make this work:

- **`codegen = "in_place"`** — heph writes the collected outputs back onto the
  source tree (write-if-changed, so an already-correct file's mtime is untouched).
- **`//@heph/introspect:outputs`** — an engine-baked magic dep. It expands, per
  declared output path, into an input that reads that path. So the target's
  *inputs are its outputs* ("format these files"), surfaced to the run script via
  `$SRC_SRC` / `$LIST_SRC_SRC`.
- **Fixed-point caching** — re-running over the already-formatted tree is a no-op
  **cache hit**, not a re-run:

  ```
  $ heph run //fmt:fmt        # second run
  ... cache 1 hit / 0 miss    # the formatter did NOT execute
  ```

### CI check: `--frozen`

Verify formatting without writing anything. Exits non-zero with a diff if a file
is not formatted:

```
$ heph run //fmt:fmt --frozen          # clean tree → passes
$ echo 'hello world' > fmt/greeting.txt
$ heph run //fmt:fmt --frozen          # dirty tree → fails
× target failed: //fmt:fmt
╰─▶ generated output differs from tree
  ╭─[diff]
  │ --- a/fmt/greeting.txt
  │ +++ b/fmt/greeting.txt
  │ @@ -1 +1 @@
  │ -hello world
  │ +HELLO WORLD
  ╰────
```

## Copy (generated): `//fmt:generated`

`codegen = "copy"` writes a generated file to the tree and **stamps** it
(`user.heph.codegen` xattr) so a later `glob()` / `file()` excludes it — it is a
build output, meant to be gitignored (see `.gitignore`). Consumers depend on the
target (`//fmt:generated`), not on the file.

```
$ heph run //fmt:generated
$ cat fmt/generated.txt
generated by //fmt:generated
```

## The distinction

| | `copy` | `in_place` |
|---|---|---|
| Meaning | generated artifact | tracked source, reformatted |
| Git | gitignored | committed |
| Globs | excluded (xattr-stamped) | visible (unstamped) |
| Inputs | its own (e.g. a schema) | its outputs (`//@heph/introspect:outputs`) |
